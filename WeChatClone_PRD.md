## 一、项目概述

### 1.1 项目背景

随着移动互联网的快速发展，即时通讯（IM）已成为人们日常生活和工作中不可或缺的基础设施。微信作为中国最大的社交平台，月活跃用户超过13亿，覆盖了社交沟通、内容分享、移动支付等多个维度。然而，在特定场景下（如企业内部通讯、垂直行业社群、隐私保护需求较高的用户群体），仍存在对类微信产品的强烈需求。

本项目旨在开发一款功能精简、体验流畅、注重隐私安全的类微信移动应用。产品核心聚焦于两大模块：**即时通讯**和**朋友圈**，通过现代化的跨平台技术栈（Flutter），实现一套代码同时运行在iOS和Android平台上，降低开发和维护成本，同时保证原生级别的性能和用户体验。

### 1.2 项目目标

1. **核心通讯能力**: 实现文本、图片、语音、视频、位置等多形态消息的实时收发，支持一对一私聊和多人群聊。
2. **社交内容分享**: 实现类似朋友圈的动态发布、点赞、评论功能，构建用户间的社交互动闭环。
3. **跨平台兼容**: 基于Flutter框架，一套代码同时支持iOS 13+和Android 8.0+设备，保持UI/UX一致性。
4. **隐私安全优先**: 端到端加密传输、本地数据加密存储、阅后即焚等隐私保护机制。
5. **高性能体验**: 消息收发延迟低于200ms（局域网环境下），首屏加载时间小于1.5秒，滑动帧率稳定在55fps以上。

### 1.3 项目范围

#### 范围内（In-Scope）

- 用户注册/登录（手机号+验证码、邮箱+密码）
- 个人资料管理（头像、昵称、个性签名、性别、地区）
- 好友系统（搜索添加好友、好友申请、通过/拒绝、黑名单）
- 一对一文字聊天（实时消息收发、已读回执、消息撤回）
- 群聊功能（创建群聊、邀请入群、群公告、群成员管理）
- 多媒体消息（图片发送/预览、语音消息录制/播放、短视频发送）
- 朋友圈（发布图文动态、点赞、评论、浏览好友动态）
- 消息通知（Push通知、Badge角标、通知栏交互）
- 账号设置（消息免打扰、深色模式、语言切换、清除缓存）

#### 范围外（Out-of-Scope）

- 微信支付/金融功能（本期不包含）
- 小程序生态（本期不包含）
- 视频号/直播功能（本期不包含）
- 公众号/订阅号体系（本期不包含）
- 城市漂流/附近的人（本期不包含）
- 桌面端/网页端（本期不包含，但API设计预留扩展能力）

## 二、产品定位与目标用户

### 2.1 产品定位

本产品定位为**"轻量级即时通讯与社交分享平台"**。与微信相比，我们做减法：去掉支付、小程序、视频号等重型模块，专注于"聊天"和"分享"两个核心场景，打造更纯粹、更轻量的社交体验。

**核心价值主张**:
- 更快的速度：精简功能带来更小的安装包（目标<50MB）和更快的响应速度
- 更好的隐私：端到端加密、阅后即焚、本地数据可控
- 更低的门槛：注册即用、无复杂设置、无广告干扰

### 2.2 目标用户画像

#### 用户群体A：年轻职场人（22-35岁）

- **特征**: 在一二线城市工作，使用智能手机频率高，日均使用社交类App超过3小时
- **痛点**: 工作消息与生活消息混杂，希望有一个纯粹的私人通讯空间；对隐私泄露敏感
- **需求**: 快速聊天、图片/文件分享、朋友圈记录生活、消息加密
- **使用场景**: 下班后与朋友群聊分享日常；在朋友圈发布旅行照片并查看好友互动

#### 用户群体B：学生群体（16-22岁）

- **特征**: 校园生活为主，社交圈以同学、社团成员为主，对新功能接受度高
- **痛点**: 现有社交工具广告多、信息噪音大；希望有更简洁的沟通工具
- **需求**: 群聊学习讨论、分享生活瞬间、语音消息便捷沟通
- **使用场景**: 班级群聊通知作业；朋友圈发布考试心得；语音消息与异地朋友聊天

#### 用户群体C：垂直社群成员（全年龄段）

- **特征**: 因共同兴趣（摄影、游戏、运动等）聚集的社群成员
- **痛点**: 微信群消息刷屏快、重要信息易被淹没；缺乏高质量的内容分享空间
- **需求**: 稳定的群聊体验、朋友圈式的沉淀式内容展示、消息置顶/标记功能
- **使用场景**: 摄影群里分享作品到朋友圈；群内重要通知用置顶功能保留

### 2.3 竞品分析

| 维度 | 本产品 | 微信 | Telegram | Line |
|------|--------|------|----------|------|
| 核心功能 | 聊天+朋友圈 | 全生态 | 聊天+频道 | 聊天+支付 |
| 安装包大小 | <50MB | >500MB | ~100MB | ~200MB |
| 端到端加密 | 支持（可选） | 不支持 | 默认支持 | 不支持 |
| 阅后即焚 | 支持 | 不支持 | 支持 | 不支持 |
| 消息撤回时限 | 2分钟 | 2分钟 | 无限 | 无限 |
| 跨平台 | iOS+Android | 全平台 | 全平台 | 全平台 |
| 广告 | 无 | 有 | 无 | 有 |
| 学习成本 | 低 | 中高 | 中 | 中 |

**差异化策略**: 以"纯净、安全、轻量"为核心卖点，吸引对微信感到臃肿、对隐私有更高要求的用户群体。

## 三、功能需求详述

### 3.1 用户模块

#### 3.1.1 注册与登录

**功能描述**: 用户通过手机号+验证码或邮箱+密码两种方式完成注册和登录。注册成功后自动生成唯一用户ID（UUID），并初始化默认个人资料。

**详细需求**:

1. **手机号注册/登录**
   - 输入11位中国大陆手机号（支持扩展其他国家号码格式）
   - 点击"获取验证码"按钮，触发短信验证码发送
   - 验证码为6位数字，有效期5分钟，同一号码60秒内只能请求一次
   - 输入验证码后自动校验，校验通过则登录/注册成功
   - 首次注册需设置昵称（2-20个字符）和头像（从相册选择或拍照）

2. **邮箱注册/登录**
   - 输入有效邮箱地址
   - 点击"发送验证邮件"，系统向邮箱发送包含4位数字验证码的邮件
   - 验证码有效期10分钟
   - 输入验证码后设置昵称和头像完成注册

3. **登录态管理**
   - 登录成功后服务端颁发JWT Token，有效期30天
   - Token过期前7天自动刷新
   - 支持多设备登录（最多5台设备同时在线）
   - 登录设备管理：可查看已登录设备列表并远程登出

4. **忘记密码**
   - 通过注册手机号或邮箱接收验证码重置密码
   - 新密码需满足：8-20位，包含字母和数字

**边界条件**:
- 手机号已被注册：提示"该手机号已绑定账号，请直接登录"
- 验证码错误：提示"验证码错误，请重新输入"
- 网络异常：提示"网络连接失败，请检查网络后重试"
- 短信发送频率限制：同一IP 24小时内最多发送10条

**异常处理**:
- 短信网关故障：降级为语音验证码（自动拨打用户手机播报验证码）
- 邮件送达延迟：提供"重发"按钮，提示"邮件可能正在送达中，请耐心等待"

---

#### 3.1.2 个人资料管理

**功能描述**: 用户可以查看和编辑自己的个人资料，包括头像、昵称、性别、地区、个性签名等信息。

**详细需求**:

1. **个人信息页**
   - 展示当前用户的完整资料
   - 点击右上角"编辑"按钮进入编辑模式
   - 支持修改的字段：头像、昵称、性别、地区、个性签名
   - 微信号（唯一ID，注册时自动生成，不可修改）
   - 二维码（绑定微信号生成，可用于扫码添加好友）

2. **头像上传**
   - 支持从相册选择或拍照
   - 提供圆形裁剪框，支持缩放和旋转
   - 上传后自动生成三尺寸缩略图：60x60（列表显示）、100x100（个人资料）、200x200（大图预览）
   - 单个文件大小限制：不超过5MB
   - 支持格式：JPG、PNG、WebP

3. **昵称规则**
   - 长度2-20个字符（中文/英文/数字/常见符号）
   - 不允许使用违规词汇（服务端敏感词过滤）
   - 同一昵称允许重复（与微信一致）

4. **个性签名**
   - 最长120个字符
   - 支持换行（最多3行）

**交互细节**:
- 修改头像时，选择图片后即时预览，确认后上传
- 昵称输入时实时校验长度和合规性，下方显示剩余字符数
- 保存修改时显示Loading状态，成功后Toast提示"修改成功"

### 3.2 好友模块

#### 3.2.1 搜索与添加好友

**功能描述**: 用户可以通过微信号、手机号或扫描二维码的方式搜索并添加好友。

**详细需求**:

1. **搜索方式**
   - **微信号搜索**: 输入对方微信号，点击搜索，展示匹配的用户卡片（头像+昵称+微信号），点击"添加到通讯录"发送好友申请
   - **手机号搜索**: 输入对方手机号，系统匹配已注册该手机号的用戶，展示用户卡片并发起好友申请
   - **二维码扫描**: 调用摄像头扫描对方个人二维码，自动识别微信号并展示添加页面

2. **好友申请流程**
   - 发送申请时可填写验证消息（最多50字，留空则使用默认消息"我想和你做朋友"）
   - 对方收到好友申请通知（红点提示+Push通知）
   - 申请详情页展示申请人头像、昵称、验证消息
   - 对方可选择"同意"或"忽略/拒绝"
   - 同意后双方互加好友，自动建立双向好友关系

3. **好友列表**
   - 按好友昵称首字母分组排序（A-Z + #（无昵称））
   - 右侧字母索引栏快速跳转
   - 支持搜索好友（按昵称/微信号模糊匹配）
   - 每个好友条目展示：头像、昵称、个性签名（截断显示）

**边界条件**:
- 搜索不到用户：提示"未找到匹配的用户"
- 已是好友：提示"你们已经是好友了"
- 自己不能添加自己：提示"不能添加自己为好友"
- 对方已将你拉黑：提示"操作失败"

#### 3.2.2 好友管理

**功能描述**: 用户可以管理好友列表，包括删除好友、设置备注名、加入/移出黑名单等操作。

**详细需求**:

1. **好友详情**
   - 点击进入好友个人主页
   - 展示：头像、昵称、微信号、性别、地区、个性签名、朋友圈封面
   - 底部操作栏：发消息（跳转聊天页面）、视频通话（预留）、删除好友、加入黑名单

2. **删除好友**
   - 确认后删除该好友关系
   - 删除后聊天记录保留（不主动清除，但无法再发送新消息）
   - 被删除方不知晓（无通知），但发消息时会收到失败提示

3. **黑名单**
   - 加入黑名单后，该用户无法给你发送消息、查看你的朋友圈
   - 黑名单中的用户会收到"消息已发出，但被对方拒收"提示
   - 支持从黑名单中移除

4. **备注名与标签**
   - 可为好友设置备注名（替代原昵称显示）
   - 支持给好友打标签（如"同事"、"同学"、"家人"），一个好友可多个标签
   - 标签管理页：创建/编辑/删除标签，查看某标签下的好友列表

### 3.3 即时通讯模块

#### 3.3.1 聊天列表

**功能描述**: 主界面为聊天列表页，展示所有聊天会话（私聊+群聊），按最后消息时间倒序排列。

**详细需求**:

1. **聊天会话条目**
   - 左侧：会话头像（私聊显示好友头像，群聊显示群头像）
   - 中间上方：会话名称（好友昵称/群名称）+ 最后一条消息预览（截断显示，最长30字符）
   - 中间下方：消息类型图标（如有未读消息时显示红色圆点+未读数）
   - 右侧：消息时间（今天显示HH:mm，昨天显示昨天，更早显示MM-dd）
   - 置顶会话：顶部显示"置顶"标签，始终排在列表最上方

2. **未读消息计数**
   - 会话条目右上角显示红色圆点+数字
   - 数字超过99显示"99+"
   - 点击进入聊天后自动清零

3. **消息类型标识**
   - 语音消息：显示喇叭图标
   - 图片消息：显示图片图标
   - 视频消息：显示播放图标
   - 撤回消息：显示"[XX撤回了一条消息]"
   - 转账消息：显示钱袋图标（预留）

4. **下拉刷新与搜索**
   - 下拉刷新：重新拉取最新的会话列表
   - 顶部搜索框：按会话名称/最后消息内容搜索
   - 搜索框右侧"新建"按钮：弹出"发起群聊"或"添加好友"选项

5. **左滑操作**
   - 左滑会话条目显示"置顶"和"删除"两个操作按钮
   - 置顶：将该会话固定在列表顶部
   - 删除：删除该会话（不删除聊天记录，如需删除聊天记录需在详情页操作）

#### 3.3.2 一对一聊天

**功能描述**: 进入与单个好友的聊天页面，支持发送和接收多种类型的消息。

**详细需求**:

1. **聊天界面布局**
   - 顶部导航栏：对方头像 + 昵称 + 右侧"更多"按钮（视频通话、个人信息、聊天文件、聊天截图、屏蔽消息、删除好友）
   - 中间消息区域：消息气泡按时间顺序排列
   - 底部输入区域：表情按钮 + 输入框 + "+"按钮（更多功能）

2. **消息气泡样式**
   - **对方发送的消息**: 左侧对齐，白色背景气泡，左侧显示对方头像（小圆形）
   - **自己发送的消息**: 右侧对齐，主题色（#6366F1）背景气泡，右侧显示自己头像（小圆形）
   - 消息时间戳：每5分钟或消息类型切换时显示居中时间标签（灰色小字）
   - 消息状态：已发送（单勾）、已送达（双勾）、已读（双勾+蓝色）

3. **文本消息**
   - 支持中英文、数字、常见标点符号
   - 最长2000字符，超出截断并提示
   - 长按消息气泡：弹出"复制"、"转发"、"收藏"、"撤回"操作菜单
   - 复制后系统剪贴板存储，其他App可粘贴

4. **图片消息**
   - 点击"+"按钮 → 选择"相册"或"相机"
   - 从相册选择：支持多选（最多9张），点击确认后压缩上传
   - 拍照：调用系统相机，拍完后可裁剪，确认后发送
   - 图片发送前预览缩略图，支持删除/替换
   - 接收方点击大图可全屏浏览，支持手势缩放、左右滑动切换

5. **语音消息**
   - 按住底部输入框右侧的"按住说话"按钮录制语音
   - 上滑取消（手指滑到上方取消区域松开则取消录制）
   - 最短1秒，最长60秒
   - 录制时显示波形动画和时长倒计时
   - 发送后对方收到语音气泡，点击播放
   - 语音播放时不支持同时播放多条（正在播放的暂停）
   - 双击加速到1.5倍速，再次恢复1倍速

6. **短视频消息**
   - 点击"+" → "拍摄视频"
   - 支持最长15秒的视频录制
   - 录制时显示倒计时和进度条
   - 发送前可预览，支持添加简单滤镜（预留）
   - 接收方点击直接播放，自动静音（除非全屏）

7. **位置消息**
   - 点击"+" → "位置" → "发送位置"
   - 调用地图SDK获取当前位置，展示在地图上
   - 可拖动地图标记调整精确位置
   - 发送后对方收到位置卡片（地图缩略图 + 地址名称 + 导航按钮）
   - 点击位置卡片可全屏查看地图，支持一键导航

8. **消息撤回**
   - 发送后2分钟内可撤回
   - 撤回后消息内容替换为"[你撤回了一条消息]"或"[XXX撤回了一条消息]"
   - 撤回操作不可逆，撤回方和被撤方均无法再查看原内容
   - 撤回时触发Push通知给对方（"XXX撤回了一条消息"）

9. **输入状态指示**
   - 对方正在输入时，顶部显示"对方正在输入..."提示
   - 基于打字频率检测（3秒内有输入行为即判定为正在输入）

#### 3.3.3 群聊功能

## 五、系统架构设计

### 5.1 整体架构

采用**分层架构 + 事件驱动**的设计模式，整体分为四层：

```
┌─────────────────────────────────────┐
│           Presentation Layer         │  ← Flutter UI (Widgets, Pages, Routes)
│         (View + ViewModel)           │
├─────────────────────────────────────┤
│           Domain Layer              │  ← Business Logic, Use Cases, Entities
├─────────────────────────────────────┤
│           Data Layer                │  ← Repositories, API Client, Local DB
├─────────────────────────────────────┤
│         External Services           │  ← WebSocket, REST API, Push Service
└─────────────────────────────────────┘
```

#### 各层职责

**Presentation Layer（表现层）**:
- 负责UI渲染和用户交互
- 使用Flutter的Widget树构建界面
- ViewModel负责状态管理和UI逻辑
- 通过Provider/Riverpod管理状态

**Domain Layer（领域层）**:
- 包含核心业务逻辑和实体定义
- Use Case封装具体的业务操作
- 不依赖任何外部库（纯Dart代码）
- 定义Repository接口，由Data层实现

**Data Layer（数据层）**:
- Repository实现类，对接Domain层的接口
- API Client负责HTTP/WebSocket通信
- 本地数据库操作（SQLite/Hive）
- 数据转换（DTO ↔ Entity）

**External Services（外部服务层）**:
- WebSocket长连接（实时消息）
- REST API（用户数据、朋友圈等）
- Push Notification Service（APNs / FCM）
- 文件存储服务（OSS/S3）

### 5.2 状态管理方案

采用**Riverpod + Freezed**组合：

- **Riverpod**: 作为Dependency Injection和状态管理的核心，替代Provider，提供更灵活的生命周期管理
- **Freezed**: 用于生成不可变的数据类和Union类型，简化State定义
- **flutter_bloc**（可选）: 对于复杂页面（如聊天页面），可采用Bloc模式管理消息流

**状态管理原则**:
- 全局状态（用户信息、登录态）使用Singleton Provider
- 页面级状态使用Family/Scoped Provider
- 组件级状态使用ConsumerWidget/local state

### 5.3 路由管理

采用**go_router**进行声明式路由管理：

```
/                    → 引导页（首次启动）/ 登录页（未登录）/ 主页面（已登录）
/login               → 登录页
/register            → 注册页
/home                → 主页面（底部Tab导航）
  /chat              → 聊天列表
  /moments           → 朋友圈
  /contacts          → 通讯录
  /settings          → 设置
/chat/:id            → 聊天页面（一对一/群聊）
/profile             → 个人资料
/profile/edit        → 编辑资料
/friend/search       → 搜索好友
/moment/publish      → 发布动态
/moment/:id          → 动态详情
```

### 5.4 网络架构

```
                    ┌──────────────┐
                    │  ApiClient    │  ← Dio HTTP Client + WebSocket
                    └──────┬───────┘
                           │
          ┌────────────────┼────────────────┐
          │                │                │
    ┌─────▼─────┐   ┌─────▼─────┐   ┌─────▼─────┐
    │ REST API  │   │ WebSocket │   │   File    │
    │ (HTTP/2)  │   │ (实时消息) │   │  Upload   │
    └───────────┘   └───────────┘   └───────────┘
```

- **REST API**: 使用Dio库，配置拦截器处理Token注入、错误统一处理、请求重试
- **WebSocket**: 使用web_socket_channel库，维护长连接处理实时消息推送
- **文件上传**: 分片上传（>10MB的文件），支持断点续传

### 5.5 本地数据存储

| 存储方式 | 用途 | 库 |
|---------|------|-----|
| SharedPreferences | 轻量配置（主题、语言、免打扰设置） | shared_preferences |
| Hive | 用户信息、Token、好友列表缓存 | hive |
| SQLite (drift) | 聊天记录本地存储、朋友圈缓存 | drift (原 Moor) |
| File System | 图片、语音、视频文件缓存 | path_provider + dio |

**数据同步策略**:
- 聊天记录：本地优先，上线后增量同步服务端
- 朋友圈：本地缓存最近50条，下拉加载时从服务端拉取
- 好友列表：登录后全量同步，之后增量更新

## 六、技术选型与理由

### 6.1 前端框架

**选型**: Flutter 3.x (Dart)

**理由**:
1. **跨平台**: 一套代码同时编译为iOS和Android原生二进制，减少50%以上的开发工作量
2. **性能**: 编译为原生ARM代码，接近原生性能，无WebView桥接开销
3. **UI一致性**: 自绘引擎（Skia），确保iOS和Android上UI完全一致
4. **热重载**: 开发时即时预览修改，大幅提升开发效率
5. **生态成熟**: 丰富的第三方包和社区支持
6. **Google背书**: 长期维护和更新，稳定性有保障

### 6.2 状态管理

**选型**: Riverpod 2.x + Freezed

**理由**:
1. **编译时安全**: Freezed生成的不可变类和Union类型在编译期检查
2. **测试友好**: Provider不依赖BuildContext，可独立于UI测试
3. **类型安全**: 全程泛型支持，避免运行时类型错误
4. **灵活性**: 支持Singleton、Family、Scoped等多种作用域

### 6.3 HTTP客户端

**选型**: Dio 5.x

**理由**:
1. **拦截器**: 内置请求/响应拦截器，方便统一处理Token、错误
2. **请求取消**: 支持CancelToken，页面销毁时自动取消未完成请求
3. **文件上传/下载**: 原生支持 multipart 和进度回调
4. **日志打印**: 开发环境可打印完整请求/响应信息
5. **自适应**: 支持JSON序列化/反序列化适配器

### 6.4 本地数据库

**选型**: Drift (原Moor)

**理由**:
1. **类型安全**: 基于Dart代码生成SQL，编译期检查SQL语法
2. **响应式查询**: 查询结果自动Observable，数据变化时UI自动更新
3. **跨平台**: 同一套代码在iOS和Android上使用相同的SQLite引擎
4. **异步API**: 全异步操作，不阻塞UI线程
5. **Migration支持**: 内置数据库版本迁移机制

### 6.5 实时通信

**选型**: WebSocket (web_socket_channel) + 自定义协议

**理由**:
1. **双向通信**: 客户端和服务端可同时发送数据，适合实时消息场景
2. **低延迟**: 保持长连接，避免HTTP轮询的延迟和开销
3. **灵活**: 可自定义消息协议，支持多种消息类型
4. **生态**: web_socket_channel是Flutter官方推荐的WebSocket库

**消息协议设计**（JSON over WebSocket）:
```json
{
  "type": "message",
  "id": "msg_uuid_v4",
  "timestamp": 1719456000000,
  "payload": {
    "senderId": "user_123",
    "receiverId": "user_456",
    "chatType": "private",
    "contentType": "text",
    "content": "你好",
    "seq": 1001
  }
}
```

### 6.6 推送服务

**选型**: Firebase Cloud Messaging (FCM) + APNs

**理由**:
1. **FCM**: Android端标准推送方案，免费、稳定、支持大数据量
2. **APNs**: iOS端必须使用Apple Push Notification service
3. **FirebaseMessaging插件**: Flutter官方插件，一套代码接入两种推送
4. **数据消息**: 支持后台接收数据消息，App自主决定通知展示方式

### 6.7 图片处理

**选型**: cached_network_image + image + flutter_image_compress

**理由**:
1. **cached_network_image**: 自动网络图片缓存、占位图、错误图
2. **image**: 图片编解码和处理（裁剪、旋转、滤镜）
3. **flutter_image_compress**: 图片压缩，控制上传文件大小

### 6.8 音频处理

**选型**: record + audioplayers

**理由**:
1. **record**: 录音功能，支持PCM/AAC格式，可控制采样率和比特率
2. **audioplayers**: 音频播放，支持并发播放、音量控制、播放进度

### 6.9 地图服务

**选型**: amap_location + amap_maps (高德地图)

**理由**:
1. **国内定位准确**: 高德地图在国内的定位精度优于Google Maps
2. **SDK成熟**: 提供位置获取、地图展示、路线规划等完整能力
3. **免费额度**: 个人开发者免费额度充足

### 6.10 文件存储

**选型**: 阿里云OSS / AWS S3

**理由**:
1. **CDN加速**: 全球CDN节点，图片/文件加载速度快
2. **按需付费**: 按存储量和流量计费，成本低
3. **API兼容**: 提供RESTful API，易于集成
4. **安全**: 支持STS临时凭证、签名URL

### 8.1 API设计规范

- **协议**: HTTPS (TLS 1.3)
- **格式**: JSON (application/json)
- **认证**: Bearer Token (JWT)
- **版本**: URL路径中包含版本号 `/api/v1/...`
- **分页**: 统一使用 `page` + `pageSize` 参数，返回 `total` + `items`
- **错误码**: 统一错误响应格式

**统一响应格式**:
```json

  "message": "success",
  "data": { ... }
```

**统一错误响应格式**:
```json

  "code": 401,
  "message": "未授权，请重新登录",

}
```

### 9.5 加载状态

- **全局Loading**: 全屏半透明遮罩 + CircularProgressIndicator（仅用于关键操作如登录、发送消息）
- **局部Loading**: 按钮内嵌Loading指示器（如"发表"按钮）
- **骨架屏**: 列表页加载时使用（如朋友圈动态列表）
- **Shimmer效果**: 图片占位符加载时使用

### 9.6 Toast与Dialog规范

#### Toast

- 位置：屏幕底部居中
- 内容：简短提示信息（不超过2行）
- 持续时间：1.5秒（成功/失败），2秒（一般提示）
- 样式：黑色半透明背景 + 白色文字 + 圆角8px + 内边距12px

#### Dialog

- 确认对话框：居中弹窗，圆角12px，阴影
- 底部弹窗（Bottom Sheet）：从底部滑入，半屏高度
- 操作按钮：主按钮（主题色），次按钮（灰色文字）
